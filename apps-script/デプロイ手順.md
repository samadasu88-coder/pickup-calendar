## 方法A：clasp で作る（推奨）

### 1. ログイン確認（上記）

### 2. プロジェクト作成

```bash
cd ~/Desktop/お迎え予定表/apps-script
clasp create --type standalone --title "お迎え予定表 同期"
```

`.clasp.json` が生成される。既存ファイル（`Code.gs`, `appsscript.json`）を上書きしないよう注意。
clasp create 直後にデフォルトの `Code.js`, `appsscript.json` ができる場合は削除して、このフォルダの2ファイルに差し替えてから push。

### 3. push

```bash
clasp push -f
```

### 4. 共有トークン（任意・推奨）を設定

Apps Scriptエディタを開く：

```bash
clasp open
```

左の「プロジェクトの設定」→「スクリプト プロパティ」→「スクリプト プロパティを追加」
- プロパティ：`SHARED_TOKEN`
- 値：適当な長い文字列（例：`pkup-2026-xxxx-yyyy`）

これを設定するとHTMLアプリ側でも同じ値を入力する必要がある。設定しなければトークンチェックなし（URLだけで叩ける）。

### 5. Web App としてデプロイ

エディタ右上「デプロイ」→「新しいデプロイ」
- 種類：「ウェブアプリ」
- 説明：任意
- 次のユーザーとして実行：自分
- アクセスできるユーザー：**全員**

「デプロイ」→ 表示される **Web App URL** をコピー。
HTMLアプリの「Apps Script Web App URL」欄に貼り付け。

### 6. 動作確認

ブラウザでWeb App URLを開いて `pickup-calendar-sync OK` が表示されれば疎通OK。
HTMLアプリで「Googleカレンダー登録」ボタンを押す。

## 方法B：ブラウザだけで作る（claspなし）

1. **アカウント確認**（上記）：script.google.com を開く前にブラウザのアカウントを切り替えておく
2. https://script.google.com を開く → 「新しいプロジェクト」
3. 既定の `Code.gs` をこのフォルダの `Code.gs` の内容で置き換え
4. メニュー「プロジェクトの設定」→「`appsscript.json` マニフェスト ファイルをエディタで表示する」を**有効化**
5. `appsscript.json` をこのフォルダの内容で置き換え
6. （任意）「スクリプト プロパティ」で `SHARED_TOKEN` を設定
7. 右上「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」
   - 実行：自分
   - アクセス：全員
8. 初回はカレンダーアクセス権の承認が必要（ここでもメインアカウントで承認すること）
9. URLをコピーしてHTMLアプリに貼る

## 修正してデプロイし直すとき

**毎回まず `./check-login.sh` でログイン確認** すること。

コード変更後：

- clasp の場合：`clasp push -f`
- ブラウザの場合：エディタで直接編集

その上で「デプロイ」→ 既存デプロイの編集アイコン → バージョン「新しいバージョン」を選択して再デプロイ。
URLは変わらない（同じデプロイIDで更新されるため）。

> **注意**：「新しいデプロイ」を選ぶとURLが変わる。基本は既存デプロイを更新する。

## トラブルシュート

- **登録先カレンダーが想定と違う**：サポアカウントで作業した可能性大。`./check-login.sh` で確認 → メインアカウントで作り直し
- **「token mismatch」**：HTMLアプリ側のトークン入力欄が空 or Apps Scriptのプロパティと不一致
- **CORSエラー**：HTMLアプリ側で `Content-Type: text/plain` を使っているか確認（標準コードはそうなっている）
- **「You do not have permission」**：Web App のアクセス権が「全員」になっているか確認
- **カレンダーIDが間違っている**：`Code.gs` の `CALENDAR_IDS` を確認
