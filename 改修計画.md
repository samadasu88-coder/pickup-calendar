# お迎え予定表 改修計画（HTMLアプリ化）

作成日：2026-05-22
※この計画をもとに、別の機会に改修する。

---

## 現状

- 毎月、勤務表が出たらお迎え表を作成している。
- 作り方：Claude Code に対し、シフトを **数字列**（パパ＝0休/1早/2遅、ママ＝0休/1出勤）で伝える。例：`11012022120...`
- Claude Code が、このフォルダの `CLAUDE.md` のルールを読んでHTMLを生成する。

## 現状の問題点

1. **入力が数字列**で、打ち間違えやすく、目で見て確認しづらい（「15日が早番か」が数字列から読めない）。
2. **毎回 Claude Code を開いて指示する**必要がある。手軽でない。

## 診断（2026-05-22）

「これを n8n で作れるか？」を検討した結果：

- **n8n は正しい道具ではない。**
- 理由：お迎え予定表の心臓部は「**複雑な独自ルール ＋ 書類（HTML）生成**」。n8n が得意なのは「サービス連携・定期実行・通知」。ルールエンジンを n8n に入れると巨大な Code ノード1個になり、n8n はただの外側にしかならない。
- 複雑なロジックは、JavaScript として書くのが自然。

## 目指す形：自己完結の HTML アプリ（1ファイル）

ブラウザで開く HTML ファイル1つに、すべてを内蔵する。**Claude Code が不要になる。**

### 構成要素

1. **入力UI** — 数字列を打つ代わりに、カレンダーのグリッドをタップして パパ＝早/遅/休、ママ＝出勤/休 を選ぶ。つーちゃん休み・じいじ休み・学校/幼稚園開始日・園休み・祝日も入力欄で。
2. **ルールエンジン** — 朝/帰りのルール、優先順位（パパ→ママ→つーちゃん→じいじ）、バス可否などを JavaScript で実装。
3. **表示** — その場でHTMLカレンダー＋施設利用サマリー＋家庭預かりサマリーを描画。
4. **印刷/保存** — 月次の予定表として出力できる形に。

### 段階（フェーズ）

- **フェーズ1**：本体（入力UI＋ルールエンジン＋表示）を HTML アプリ化。
- **フェーズ2**：Googleカレンダー連携（本体が安定してから。ここは別途、スクリプトや n8n が絡む余地あり）。

## 改修時に詰めること

- ルールの最新版は、このフォルダの `CLAUDE.md` と `マニュアル.md` が正。改修時はまずこれを読む。
- 入力データの保存：毎月の入力を localStorage に残すか、前月をコピーして使えるようにするか。
- スマホで入力・閲覧しやすいレイアウト。
- 既存の色分け（パパ青/ママピンク/つー緑/じいじオレンジ）を踏襲。
